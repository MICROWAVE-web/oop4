# Умная музыкально-справочная колонка

Лабораторная работа №4: паттерны **Делегирование** и **Proxy**.  
Голосовой ассистент с поддержкой музыки, справок и новостей.

## Запуск

```bash
python3 main.py
```

В режиме офлайн-конфигурации (по умолчанию) ввод имитирует голос: вводите команды текстом в консоль.

---

## Возможные команды

Команды распознаются модулем NLU (при использовании `RuleBasedNLU`). Достаточно, чтобы в фразе встречалось ключевое слово.

### Музыка

| Команда | Примеры |
|--------|---------|
| Включить музыку | `включи музыку`, `включи песню`, `включи трек`, `поставь музыку` |

Ассистент ищет треки через выбранный музыкальный сервис и воспроизводит первый результат.

---

### Справка (информация)

| Команда | Примеры |
|--------|---------|
| Любой вопрос без ключевых слов | `какой сегодня праздник`, `что такое Python`, `кто такой Пушкин` |

Ответ формируется выбранным справочным сервисом (Wikipedia, Yandex, Google и т.д.).

---

### Новости

| Команда | Примеры |
|--------|---------|
| Новости по теме | `новости`, `новости спорт`, `новости политика` |

Тема — часть фразы после слова «новости». Сервис возвращает список новостей (если реализация это поддерживает).

---

### Громкость

| Команда | Примеры |
|--------|---------|
| Установить громкость (0–10) | `громкость 7`, `громкость 5`, `поставь громкость 3` |

Если число не указано, используется значение по умолчанию (5).

---

### Выход

| Команда | Примеры |
|--------|---------|
| Завершить работу | `выключись`, `стоп` |

Ассистент прощается и завершает программу.

---

## Конфигурации

В `main.py` заданы три конфигурации:

- **dialog_offline** — локальная, офлайн (Wikipedia, локальные файлы, консольный ввод).
- **dialog_smart** — облачная (Spotify через Proxy, Google, Bluetooth, нейросети).
- **dialog_car** — гибридная (VK Music, Yandex, потоковый плеер).

Для запуска другой конфигурации измените последние строки в `main.py` (раскомментируйте нужный `run()`).

## Структура проекта

| Файл | Назначение |
|------|------------|
| `interfaces.py` | Абстрактные интерфейсы сервисов |
| `music_services.py` | Музыкальные сервисы и ProxyMusicService |
| `info_services.py` | Справочные сервисы |
| `player.py` | Плееры |
| `asr.py` | Распознавание речи |
| `nlu.py` | Понимание намерений (NLU) |
| `tts.py` | Синтез речи |
| `voice_interfaces.py` | Голосовой ввод/вывод (консоль, микрофон) |
| `dialog_manager.py` | Диалоговый менеджер |
| `main.py` | Точка входа и конфигурации |

## Структура проекта
    ASR -> NLU -> логика -> сервисы -> TTS -> Player
